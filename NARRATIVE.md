# AI 기반 인프라 보안 스캐너 MVP — 핵심 네러티브

**추출일**: 2026-06-24  
**기반 문서**: [AI 기반 인프라 보안 스캐너 MVP PRD](./prd.md)

## 핵심 네러티브

리눅스 서버 인프라의 보안 점검을 담당하는 보안팀은 매일 업데이트되는 CVE를 수동으로 모니터링하면서 취약점 발견이 늦어지고, 주요정보기반시설 보안 가이드에 따른 수동 점검은 시간이 오래 걸리고 오류 가능성이 높다는 문제에 직면해 있습니다. 특히 인프라가 수백~수천 대의 서버로 확장되면서 수동 관리는 현실적으로 불가능해졌습니다. 

이 시스템은 **"보안 점검을 완전히 자동화할 수 있다"**는 관점에서, Ansible로 대규모 서버의 정보를 동시에 수집하고, Claude AI를 통해 지능형 취약점 분석과 조치 방안을 제시하며, 웹 대시보드로 시각화함으로써 문제를 해결합니다. 

결과적으로 보안담당자는 반복적인 수작업에서 해방되고, 대규모 인프라 전체에서 빠르게 취약점을 찾아 우선순위를 정하고 경영진에게 명확한 보안 상태를 보고할 수 있게 됩니다.

---

## 구성 요소 분석

### 대상 (Audience) [명시]
리눅스 서버 인프라의 보안 점검을 담당하는 보안팀과 보안담당자

- 매일 수동으로 CVE 모니터링 중
- 대규모 인프라 전체의 보안 현황을 파악해야 함
- 주요정보기반시설 보안 규정 준수 필요

---

### 진짜 문제 (Root Problem) [명시]

**표면 요구**: "보안 점검을 빠르게 하고 싶다"  
**근본 원인**: 수동으로 이루어지는 보안 점검 체계

**구체적 문제들**:
1. **취약점 탐지 지연** — CVE는 매일 업데이트되지만 수동 모니터링으로는 발견이 늦음
2. **높은 오류 가능성** — 주요정보기반시설 보안 가이드에 따른 수동 점검이 비효율적이고 인간 실수 발생
3. **규모 확장 불가** — 수백~수천 대의 서버를 수동으로 관리하는 것은 현실적으로 불가능
4. **자동화 도구 부재** — 취약점을 빠르게 찾고 보고할 수 있는 자동화된 도구가 없음

**이로 인한 영향**:
- 심각한 보안 위협에 빠르게 대응하지 못함
- 인프라 전체에 대한 불완전한 보안 커버리지
- 보안팀의 반복적이고 비효율적인 수작업 증가
- 경영진에 대한 보안 상태 현황 보고 및 추적 어려움

---

### 맥락·타이밍 (Context) [명시]

**시장·기술 변화**:
- CVE가 매일 업데이트되는 빠르게 변하는 위협 환경
- 인프라 규모가 점점 확장되고 있음 (수십 대 → 수백 대 이상)
- 주요정보기반시설 보안 규정이 점점 강화되고 있음

**조직 변화**:
- 인프라 관리 대상이 증가하면서 보안팀의 업무 부담이 급증
- 경영진의 보안 상태 현황 보고 및 추적에 대한 요구 증가

**왜 지금인가**:
인프라 규모가 커졌지만 점검 방식은 여전히 수동이라는 괴리가 심해지고 있음. 자동화 없이는 더 이상 스케일할 수 없는 상황에 직면함.

---

### 핵심 관점·베팅 (Core Hypothesis) [명시]

**이 제품이 거는 가설**:
> "보안 점검을 완전히 자동화하고, AI 기반 분석으로 정확한 인사이트를 제공할 수 있다면, 보안팀의 효율성을 획기적으로 높일 수 있다."

**구체적 베팅**:
1. **자동화 가능성** — Ansible을 통해 대규모 서버의 정보를 동시에 안정적으로 수집할 수 있다
2. **AI의 정확성** — Claude AI를 통한 지능형 분석이 인간의 실수를 줄이고 더 정확한 취약점 분류와 조치 방안을 제시할 수 있다
3. **시각화의 효과** — 대시보드를 통한 직관적 시각화가 의사결정 속도와 우선순위 판단을 개선한다

**다른 접근과의 차별성**:
- 단순 자동화 도구가 아니라, 지능형 AI 분석과 결합
- 한국어 조치 방법으로 팀원의 이해도와 실행 속도 향상
- 대규모 병렬 처리 (5~10개 서버 동시 점검)로 스케일 확보

---

### 해결 방향 (Solution Approach) [명시]

**3가지 핵심 기능**:

1. **자동화된 서버 정보 수집** (Ansible)
   - SSH 기반 원격 명령 실행
   - OS 정보, 패키지, 보안 설정, 실행 중인 서비스 수집
   - 민감한 정보는 자동으로 마스킹

2. **AI 기반 취약점 분석** (Claude API)
   - 수집한 서버 정보와 NVD 취약점 데이터를 연동
   - Claude AI가 영향받는 CVE, 심각도, 한국어 조치 방법 제시
   - 패키지 버전과 취약점을 자동으로 매칭

3. **웹 대시보드 시각화** (React)
   - 서버 상태 현황 (정상/경고/위험)
   - 취약점 심각도별 분포 시각화
   - 서버별·심각도별 필터링 및 상세 정보 조회
   - 자동 새로고침으로 최신 정보 제공

**기술 스택**:
- 백엔드: Python FastAPI
- 프론트엔드: React
- 데이터베이스: PostgreSQL
- 정보 수집: Ansible
- AI 분석: Claude API
- 취약점 데이터: NVD API

---

### 성공의 모습 (Outcome) [명시]

**사용자 여정**:
1. CSV 파일로 5~10개 리눅스 서버 등록
2. "점검 시작" 버튼 → 자동으로 모든 서버 정보 수집 (병렬 처리)
3. AI가 자동으로 취약점 분석 및 조치 방법 제시
4. 대시보드에서 심각도별 필터링 및 영향받는 서버 확인
5. 경영진에게 명확한 보안 상태 보고서 제공

**정량적 효과**:
- 수동 점검 시간: 수시간 → 수십 분 (80% 이상 감소)
- 점검 빈도: 월 1~2회 → 필요 시 즉시 (자동화로 수행 가능)
- 커버리지: 수십 대 서버 → 수백~수천 대 (확장성 확보)
- 오류율: 인간 실수 포함 → AI 분석으로 감소

**정성적 효과**:
- 보안담당자는 반복적 수작업에서 해방되고 전략적 업무에 집중 가능
- 경영진은 실시간 보안 상태를 명확히 이해 가능
- 팀 전체의 보안 의식 및 대응 속도 향상
