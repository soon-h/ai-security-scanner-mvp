## 📊 팀 역량 분석

### 강점
✅ **팀의 "무엇"은 명확하다**
- OS 정보 추출 → AI 분석 → 대시보드 표시가 핵심
- 이 순서가 결정되면 나머지가 따라옴

✅ **문제와 사용자가 구체적이다**  
- 현장의 비효율("수동으로 매일 스캔")을 파악
- 사용자 역할 정의(Admin/Operator/Viewer)가 정확

✅ **동기 협업 체제**  
- 9~18시 같은 시간에 일함 → 즉시 피드백 가능
- 막히면 바로 도움 요청 가능

### 약점
⚠️ **초심자 3명 + 회사 업무 병행**
- 예상 실제 가용 시간: 20일의 **50~60%** (10~12일 수준)
- Ansible, React 등 새로운 기술 학습 병행
- 학습 + 구현 = 시간 대폭 증가

⚠️ **AI API 활용 방안이 가장 불명확**
- "AI가 자동으로 리뷰한다" = 무엇을 의미하는가?
  - 취약점을 자동 분류? 조치 방안을 자동 생성?
  - 단순히 필터링? 스코어링?
- 이게 명확해야만 프롬프트, 비용, 일정이 결정됨

⚠️ **기술 난이도와 예상 시간의 불일치**
- Ansible: "3일"인데, 초보자가 처음 배우면 **4~6일 소요 가능**
- React 대시보드: "3일"인데, 초보자가는 **6~10일 이상 소요 가능**
- 버전 매칭: "정확도"가 가장 예측 어려운 부분

## 🚨 AI API 활용 방안 - 즉시 결정 필요

### 현재 상태: **미결정** ⛔

### 질문: "AI가 자동으로 리뷰한다"는 정확히 무엇을 의미하는가?

| 옵션 | 의미 | 프롬프트 복잡도 | 비용 | 예상 시간 |
|------|------|--------|------|---------|
| **A** | 취약점을 **자동 분류**<br/>(심각도, 패턴별) | 낮음 | 낮음 | 1~2일 |
| **B** | **조치 방안을 자동 생성**<br/>(어떻게 고칠지) | 중간 | 중간 | 2~3일 |
| **C** | **두 가지 모두** | 높음 | 높음 | 3~4일 |
| **D** | **필터링만**<br/>(사람이 리뷰할 것 자동 분류) | 낮음 | 낮음 | 1일 |

### 프롬프트 예시 (옵션 B 기준)

```python
prompt = """

시스템 정보:
{system_info}

취약한 패키지:
{vulnerable_packages}

응답 형식 (JSON):

  "vulnerabilities": [
```
