## What to build

여러 서버를 안정적으로 점검하기 위한 기능을 구현합니다. 5~10개 서버를 동시에 처리하여 점검 시간을 단축합니다. 네트워크 오류 등으로 인한 일시적 실패는 자동으로 재시도(최대 3회, 30초 간격)하고, 응답 없는 서버는 5분 후 자동으로 중단됩니다. 실패한 서버는 상세한 오류 정보를 기록하고, 성공한 서버는 계속 진행합니다. React 대시보드에서 점검 진행 상황을 실시간으로 표시합니다.

**핵심 요소:**
- 점검 실행 API (`POST /api/servers/scan` 또는 `POST /api/scans`)
- 비동기 병렬 처리 (동시 5~10개, 설정 가능)
- 자동 재시도 (3회, 30초 간격)
- 서버당 타임아웃 (5분)
- 실패 서버 상세 오류 기록
- 점검 상태 조회 API
- React에서 점검 진행 상황 실시간 표시
- 점검 완료 후 성공/실패 요약

## Acceptance criteria

- [ ] 점검 시작 API 구현
- [ ] 동시 처리 구현 (asyncio, ThreadPoolExecutor 등 사용)
- [ ] 동시 처리 수 설정 가능 (기본값: 5~10)
- [ ] 자동 재시도 구현 (3회, 30초 간격)
- [ ] 타임아웃 구현 (5분)
- [ ] 실패 서버 상세 오류 기록 (연결 오류, 권한 오류 등)
- [ ] 점검 상태 조회 API (진행 중, 완료, 실패)
- [ ] React 대시보드에서 점검 진행 상황 표시
- [ ] 점검 완료 후 요약 (성공/실패 서버 수)
- [ ] 단위/통합 테스트: 동시 처리, 재시도, 타임아웃 검증
