## What to build

서버 접근에 필요한 계정 정보(사용자명, 비밀번호, SSH 키)를 안전하게 저장하고 관리합니다. AES-256 암호화를 사용하며, 암호화 키는 환경변수 `INFRA_SECURITY_MASTER_KEY`에서 로드합니다. 계정 정보는 저장 시 암호화되고, 점검 중에만 메모리에서 복호화됩니다. 관리자만 접근할 수 있으며, 로그나 오류 메시지에는 민감 정보가 노출되지 않습니다.

**핵심 요소:**
- AES-256 암호화/복호화 함수 구현
- 환경변수에서 마스터 키 로드
- 데이터베이스에 암호화된 형태로 저장
- 관리자 권한 검증
- 로그/오류 메시지 필터링
- 암호화 키 로테이션 고려

## Acceptance criteria

- [ ] AES-256 암호화/복호화 함수 구현
- [ ] 환경변수 `INFRA_SECURITY_MASTER_KEY` 로드 및 검증
- [ ] 계정 정보 저장 시 자동 암호화
- [ ] 점검 실행 시에만 메모리에서 복호화
- [ ] 계정 정보 조회 API에 관리자 권한 검증
- [ ] 로그 기록 시 민감 정보 마스킹 (계정명, 비밀번호, 키)
- [ ] 오류 메시지에 민감 정보 노출 방지
- [ ] 단위 테스트: 암호화/복호화 검증
