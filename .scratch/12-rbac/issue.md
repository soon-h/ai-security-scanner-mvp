## What to build

역할 기반 접근 제어(RBAC)를 구현하여 사용자의 권한을 관리합니다. 두 가지 역할을 지원합니다: 관리자(Admin)는 모든 서버와 계정 정보에 접근 가능하고 점검을 실행할 수 있으며, 뷰어(Viewer)는 담당 팀/부서의 서버와 취약점만 읽기 전용으로 볼 수 있습니다. 모든 API 엔드포인트에 권한을 검증하고, 대시보드도 권한에 따라 필터링됩니다.

**핵심 요소:**
- 사용자 역할 정의 (Admin, Viewer)
- 팀/부서 정보 저장
- API 엔드포인트 권한 검증 (데코레이터)
- 데이터 필터링 (권한별 서버 목록 반환)
- 대시보드 권한 기반 표시
- 사용자 관리 페이지 (관리자만)

## Acceptance criteria

- [ ] 사용자 역할 추가 (admin/viewer)
- [ ] 팀/부서 정보 저장 및 관리
- [ ] API 권한 검증 데코레이터 구현
- [ ] 관리자: 모든 서버 접근, 점검 실행, 계정 정보 조회
- [ ] 뷰어: 담당 서버만 조회, 점검 실행 불가, 계정 정보 조회 불가
- [ ] API 응답 필터링 (권한에 따라 데이터 반환)
- [ ] 대시보드 UI 권한 기반 표시 (점검 버튼, 계정 정보 등)
- [ ] 사용자 관리 페이지 (관리자용)
- [ ] 권한 없는 요청 처리 (403 Forbidden)
- [ ] 단위/통합 테스트: 권한 검증 및 데이터 필터링
